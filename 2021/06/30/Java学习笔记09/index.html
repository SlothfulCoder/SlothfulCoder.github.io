Java学习笔记09

发表于 2021-06-30 更新于 2021-07-01

关键字：

1.集合框架体系

集合主要是两组（单列集合，双列集合），Collection接口有两个重要的子接口List、Set，他们的实现子类都是单列集合，Map接口的实现子类是双列集合，存放K-V

2.Collection接口

Collection接口没有直接实现子类，是通过它的子类接口Set和List来实现的，其中List是有序的，Set是无序的，有些可以存放重复元素，有些不可以。

Collection接口和常用方法

以实现子类ArrayList演示

  1. add：添加单个元素

  2. remove：删除指定元素

  3. contains：查找元素是否存在

  4. size：获取元素个数

  5. isEmpty：判断是否为空

  6. clear：清空

  7. addAll：添加多个元素

  8. containsAll：查找多个元素是否都存在

  9. removeAll：删除多个元素

遍历集合

  1. Iterator迭代器

    Iterator对象称为迭代器，主要用于遍历Collection集合中的元素，Collection接口继承了Iterator接口，所以Collection下的所有实现子类都可以使用迭代器

    迭代器用法示例：

    迭代器分析

    1. Collection子类实例调用iterator()方法得到迭代器
    2. 迭代器调用hasNext方法判断是否还有下一个元素
    3. 迭代器调用next方法，指针下移，并将下移后位置上的元素返回
    4. 遍历结束后，next指向最后一个元素，再次调用会抛出异常，要再次使用需要先重置迭代器
    5. 迭代器本身本身并不存放对象，仅用于遍历集合

  2. 增强for

    增强for就是简化版的迭代器，且底层仍是迭代器，只能用于遍历集合或数组

  3. 普通循环

3.List接口

List接口时Collection接口的子接口

  1. List集合类中的元素有序（即添加顺序和取出顺序一致）、且可重复
  2. List集合中的每一个元素都有其对应的顺序索引，即支持索引
  3. List常用实现类有：ArrayList、LinkedList和Vector

List常用方法

  1. add：在index位置插入元素

  2. addAll：从index位置开始将集合中的所有元素添加进来

  3. get：获取指定index位置的元素

  4. indexOf：获取元素第一次出现索引

  5. lastIndexOf：获取元素最后一次出现索引

  6. remove：移除指定索引元素

  7. set：修改索引元素

  8. subList：获取[2,4)的子集合
