4.封装

定义：封装就是把抽象的数据【属性】和对数据操作【方法】封装在一起，数据被保护在内部，程序的其他部分只有通过被授权的操作【方法】，才能对数据进行操作。

实现步骤：

  1. 将属性私有化private
  2. 提供公共（public）的set方法，用于对属性判断并赋值
  3. 提供（public）的get方法，用于获取属性的值

注意：我们可以将set方法写在构造器中，这样仍然可以使初始化数据受保护。

5.继承

继承可以解决代码复用，子类继承父类中的所有属性和方法，减少大量冗余代码。

父类又叫超类、基类；子类又叫派生类

基本语法

注意点：

  1. 子类继承了所有属性和方法，但私有属性和方法不能在子类中直接访问，要通过公共的方法去访问。
  2. 当创建子类对象时，默认情况下总会去调用父类的无参构造器，如果父类没有无参构造器，则必须在子类的构造器中用super去指定使用父类的哪个构造器完成对父类的初始化工作，否则编译不会通过。
  3. super指定构造器方法：super(参数列表)
  4. super指定构造器时，必须放在子类构造器的第一行。（super指定构造器只能写在子类构造器中）
  5. super()和this()都只能放在构造器第一行，因此这两个方法不能共存在一个构造器。
  6. Java所有类都是Object类的子类，Object是所有类的基类。
  7. 父类构造器的调用不限于直接父类！将一直往上追溯到Object类（顶级父类）
  8. Java中是单继承机制，即一个子类最多只能直接继承一个父类

属性/方法访问规则：

  1. 首先看子类是否有该属性
  2. 如果子类有这个属性，并且可以访问，则返回信息
  3. 如果子类没有这个属性，就看父类有没有这个属性（如果父类有该属性，并且可以访问，就返回信息……）
  4. 如果父类没有该属性，就按照（3）的规则，继续找上级父类，直到Object。

6.super

super代表父类的引用，用于访问父类的属性、方法、构造器。super访问父类属性和方法时不能访问private属性；super访问父类构造器时，只能出现在子类构造器的第一句，且只能出现一句。

特别作用：可以绕过本类，直接访问父类的重名属性或方法。

补充：上面知识点继承有对super的描述，这里不重复写了。

7.Override——重写

方法重写也叫方法覆盖，需要满足下面的条件

  1. 子类的方法的参数、方法名称，要和父类方法的参数、方法名称完全一样。
  2. 子类方法的返回类型和父类方法返回类型一样，或者是父类返回类型的子类
  3. 子类方法不能缩小父类方法的访问权限

注意：private、static、final方法不能重写

8.多态

多态基本写法

多态可以提高代码复用性，有利于维护，比如，定义一个父类作为参数的方法，这个可以接收任何所有子类，不必为每一种类重载这个方法。

初识多态

  1. 一个对象的编译类型和运行类型可以不一致。
  2. 编译类型在定义对象时，就确定了，不能改变。
  3. 运行类型是可以变化的。
  4. 编译类型看定义时 = 号的左边，运行类型看 = 号的右边。

向上转型

最常见的多态形式，本质是父类的应用指向了子类的对象。

语法

特点：

  1. 可以调用父类中的所有成员（需遵守访问权限）
  2. 不能调用子类中的特有成员。（因为在编译阶段，能调用哪些成员，是由编译类型决定的）
  3. 最终运行效果看运行类型的具体实现，即在运行时，按照从运行类型开始查找方法。

注意属性没有重写之说！属性的值看编译类型

finalize方法

当对象被回收时，系统自动调用对象的finalize方法，子类可以重写该方法，做一些释放资源的操作。

那么对象什么时候被回收呢？当某个对象没有任何引用时，则jvm就认为这个对象是一个垃圾，就会使用垃圾回收机制来销毁对象，在销毁对象前，会先调用finalize方法。

垃圾回收机制的调用，是由系统来决定（即有自己的GC算法），也可以通过System.gc()主动触发垃圾回收机制。
