1.正则表达式入门

正则表达式：regular expression => RegExp ，一个正则表达式，就是用某种模式去匹配字符串的一个公式。

解释

在Matcher中有一个int类型的groups数组，find方法会根据正则表达式去匹配字符串，如果匹配到，就把匹配到的字符串在content中的起始索引存放到groups数组的0号元素，结束索引+1放到1号元素。

下面是Matcher的group方法，给group方法传参0，可以根据0号和1号元素中的起始索引和结束索引+1在content中截取匹配到的字符串

再次调用find会继续匹配下一个符合正则表达式的字符串，同样把索引存放到groups数组的0号和1号元素（覆盖掉上一次的）

添加括号代表分组，按顺序把第一组的索引存放到2号和3号元素，第二组的索引存放到4号和5号元素……

比如：匹配到1998

groups[0] = 0, // 1的索引

groups[1] = 4, // 8的索引+1

groups[2] = 0, // 1的索引

groups[3] = 2, // 第一个9的索引+1

groups[4] = 2, // 第二个9的索引

groups[5] = 4, // 8的索引+1

3.正则表达式三个常用类

java.util.regex 包主要包括三个类：Pattern类、Matcher类、PatternSyntaxException类

Pattern对象是一个正则表达式对象，Pattern类没有公共构造方法，要创建一个Pattern对象，调用其公共静态方法compile，它返回一个Pattern对象。

Matcher对象是对输入字符串进行解释和匹配的引擎，与Pattern类一样，Matcher也没有公共构造方法。你需要调用Pattern对象的matcher方法获取一个Matcher对象

PatternSyntaxException是一个非强制异常类，它表示一个正则表达式模式中的语法错误。

4.反向引用

问题引出：如果我们想要匹配abbc格式的数字（如1998），就必须用到反向引用

反向引用：我们用圆括号对匹配到内容分组，分组编号从左向右，编号依次是1，2，3……这些分组的内容可以被引用，形成更使用的匹配模式。

这种引用既可以是在正则表达式内部，也可以是在正则表达式外部

内部反向引用：\\分组号

外部引用：$分组号
