1.AOP参数

上面代码中，方法test2有两个参数，JoinPoint类型的JoinPoint类型的joinPoint和Object类型的ret

JoinPoint对象

JoinPoint对象封装了SpringAop中切面方法的信息，在切面方法中添加JoinPoint参数，就可以获取到封装了该方法信息的JoinPoint对象

常用方法

  1. Signature getSignature()：获取封装了署名信息的对象，在该对象中可以获取到目标方法名，所属类的Class等信息
  2. Object[] getArgs()：获取传入目标方法的参数对象
  3. Object getTarget()：获取被代理的对象
  4. Object getThis()：获取代理对象

Signature对象

Signature对象是JoinPoint对象调用getSignature()方法的返回值，该对象可以获取到目标方法名，所属类的Class信息

常用方法
  1. String getName()：获取目标方法名
  2. Class getDeclaringType()：获取目标方法所属类的class对象
  3. String getDeclaringTypeName()：获取目标方法所属类的全限定类名
  4. int getModifiers()：获取目标方法声明类型，返回值是int类型，配合Modifier.toString()使用
  1. Method getMethod()：获取目标方法对象

4.多个切面执行顺序

有时候一个方法会被多个切面共同切，这种情况下，如果两个切面都有前置通知，那么这两个前置通知必然会有一个执行顺序，那么这个顺序的规则是什么呢？

默认规则

首先，切面的执行顺序不是一个排序，而是一个环状层级关系，层级的排序是按照切面类的类名按字符串排序的，a-z，字母小的在外层。

修改层级顺序

在切面类上加上**@order(int value)**注解可以修改切面类的层级位置，value的值越小，越在外层，value的默认值是一个非常大的数2147483647

实际传入参数和实际返回值

实际传入参数由最里层的环绕通知（前）决定，实际的返回结果由最外层的环绕通知（后）决定
