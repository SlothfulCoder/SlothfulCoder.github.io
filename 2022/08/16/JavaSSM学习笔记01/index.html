1.Spring简介

Spring是分层的 Java SE/EE应用 full-stack 轻量级开源框架，以 IoC（Inverse Of Control：反转控制）和 AOP（Aspect Oriented Programming：面向切面编程）为内核。

提供了展现层 SpringMVC 和持久层 Spring JDBCTemplate 以及业务层事务管理等众多的企业级应用技术，还能整合开源世界众多著名的第三方框架和类库，逐渐成为使用最多的Java EE 企业应用开源框架。

2.Spring入门

在我们使用Spring之前，通常在Service层需要用到Dao层的对象，就需要在Service层创建Dao对象，这样Dao层的类就和Service层的类绑定了，耦合度较高

使用Spring框架，把Dao层的类的全限定类名写到配置文件，由Spring框架读取配置文件，用反射创建实例；需要对象实例时就从Spring中获取，从而降低耦合度，即把创建对象的工作交给Spring来做，使用对象时就从Spring中获取

3.Spring配置文件

bean标签属性

bean标签在beans标签下，一个bean标签对应一个全限定类名，全限定类名可重复

  1. id：bean实例在Spring容器中的唯一标识，自定义值

  2. class：bean的全限定类名

  3. scope：指对象的作用范围

  4. init-method：指定类中的初始化方法名称，指定的方法，会在实例创建后自动执行

  5. destroy-method：指定类中的销毁方法名称，指定的方法，会在实例被销毁前自动执行

  6. factory-method：指定创建实例方法

    • 无参构造器：不写这个属性，默认由Spring调用无参构造器创建实例
    • 工厂类静态方法：class属性的值为工厂类，factory-method属性的值为工厂类创建实例的静态方法
  • 工厂类非静态方法：先引入工厂类实例，再写一个bean标签用factory-method指定工厂类创建实例的非静态方法

依赖注入

依赖注入（Dependency Injection）：是 Spring 框架核心 IOC 的具体实现。

在编写程序时，通过控制反转，把对象的创建交给了 Spring，但是代码中不可能出现没有依赖的情况。IOC 解耦只是降低他们的依赖关系，但不会消除。例如：业务层仍会调用持久层的方法。那这种业务层和持久层的依赖关系，在使用 Spring 之后，就让 Spring 来维护了。简单的说，就是坐等框架把持久层对象传入业务层，而不用我们自己去获取。

本质就是在Spring中，创建实例的同时，给这个实例的中指定的属性赋值。比如：Service和Dao都在Spring容器中，而最终程序直接使用的是Service，所以可以在Spring容器中，将Dao设置到Service中的dao属性，这样只获取service的实例即可

4.Spring相关API

ApplicationContext接口

ApplicationContext是接口，常用的有三个实现类

  1. ClassPathXmlApplicationContext：它是从类的根路径下加载配置文件，构造器参数是main下的resource文件夹中的xml配置文件名
  2. FileSystemXmlApplicationContext：它是从磁盘路径上加载配置文件，配置文件可以在磁盘的任意位置，构造器参数是磁盘中xml配置文件的绝对路径
  3. AnnotationConfigApplicationContext：当使用注解配置容器对象时，需要使用此类来创建 spring 容器。它用来读取注解

getBean()方法

getBean方法获取容器中的实例

  1. Object getBean(String name)：参数是读取到的配置文件中bean标签的id
  2. T getBean(Class<T> requiredType)：参数是类的Class对象，表示根据类型从容器中匹配Bean实例，当容器中相同类型的Bean有多个时，则此方法会报错。
