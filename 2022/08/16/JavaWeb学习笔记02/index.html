JavaWeb学习笔记02

发表于 2022-08-16 更新于 2021-09-05

关键字：Tomcat、Servlet

1.web相关概念

软件架构

  1. C/S：客户端/服务器端
  2. B/S：浏览器/服务器端

网络通信三要素

  1. IP：电子设备(计算机)在网络中的唯一标识。

  2. 端口：应用程序在计算机中的唯一标识。 0~65536

  3. 传输协议：规定了数据传输的规则

  • tcp:安全协议，三次握手。 速度稍慢

  • udp：不安全协议。 速度快

资源分类

  1. 静态资源：所有用户访问后，得到的结果都是一样的，称为静态资源，静态资源可以直接被浏览器解析。如：html、css、JavaScript
  2. 动态资源：每个用户访问相同资源后，得到的结果可能不一样，称为动态资源。动态资源被访问后，需要先转换为静态资源，再返回给浏览器。如servlet/jsp、php、asp

2.web服务器软件

服务器软件：可以接受用户请求，并处理请求，做出相应的软件

服务器：安装了服务器的软件的计算机

web服务器软件：一样可以处理用户请求，在web服务器软件中，可以部署web项目，让用户通过浏览器访问这些项目

JavaEE：Java语言在企业级开发中使用的技术规范的总和，一共规定了13项大的规范

常见Java相关的web服务器软件

  1. weblogic：oracle公司，大型的JavaEE服务器，支持所有JavaEE规范，收费的
  2. webSphere：IBM公司，大型的JavaEE服务器，支持所有JavaEE规范，收费的
  3. JBOSS：JBOSS公司，大型的JavaEE服务器，支持所有的JavaEE规范，收费的
  4. Tomcat：Apache基金组织，中小型的JavaEE服务器，仅仅支持少量的JavaEE规范servlet/jsp，开源的，免费的

3.Tomcat

下载

http://tomcat.apache.org/

安装

解压压缩包即可，注意安装目录建议不要有中文和空格

启动

双击安装目录下的bin/startup.bat即可，在浏览器输入http://localhost:8080访问自己，访问别人的把localhost换成ip地址

可能出现的问题

  1. 黑窗口一闪而过

    • 原因：没有配置正确的JAVA_HOME环境变量，tomcat会根据环境变量JAVA_HOME寻找jdk
    • 解决方法：配置JAVA_HOME环境变量

  2. 启动报错

    • 原因：8080端口被占用

    • 解决方法一：用 netstat -ano 命令找到占用8080端口程序的uid，在任务管理器根据uid关闭掉这个程序

    • 解决方法二：修改tomcat的端口，在conf/server.xml中修改

把端口修改成80，访问时就不用输入端口号了，因为80端口号是http协议的默认端口号

关闭

  1. 正常关闭：两种方法
    • bin/shutdown.bat
    • ctrl+c
  2. 强制关闭：关闭黑窗口

配置

三种配置方法，不过一般用第三种

第一种

直接把项目文件夹放入webapps文件夹，或者将项目打包成war包，放入webapps文件夹（war包会自动解压）

访问新添加的项目，浏览器地址中加上项目名（虚拟目录）和文件名，比如要访问的是/webapps/hello/hello.html，输入的地址就应该是http://localhost:8080//hello/hello.html

第二种

5.Servlet

Servlet（server applet）意思是运行在服务器端的小程序，Servlet就是一个接口，定义了Java类被浏览器访问到(tomcat识别)的规则。将来我们自定义一个类，实现Servlet接口，复写方法。

Servlet入门

  1. 创建Java EE项目
  2. 定义一个类，实现Servlet接口
  3. 实现接口中的抽象方法
  4. 在WEB-INF/web.xml中配置Servlet（Servlet3.0之后可以用注解）
  5. 每次访问虚拟目录，实现的service方法就会被调用一次

执行原理

  1. 当服务器接受到客户端浏览器的请求后，会解析请求URL路径，获取访问的Servlet的资源路径
  2. 查找web.xml文件，是否有对应的<url-pattern>标签体内容。
  3. 如果有，则在找到对应的<servlet-class>全类名
  4. tomcat会将字节码文件加载进内存，并且创建其对象
  5. 调用其方法

注解配置资源路径

使用web.xml配置指定Servlet的位置，每创建一个servlet就需要在web.xml中添加一对标签，如果有大量的servlet就会很麻烦；在Servlet3.0（Java EE 6）之后，可以用注解实现这个功能，不再需要配置web.xml文件来指定Servlet的位置

用法非常简单，只需要在Serlet实现类上使用@WebServlet注解，进行配置即可

资源路径写法

  1. /xxx：路径匹配
  2. /xxx/xxx:多层路径，目录结构
  3. *.do：扩展名匹配
  4. /*：任意路径

优先级：如果匹配不到1、2才会匹配4，4的优先级大于3，即有4存在，3就不会匹配到

一个Servlet可以对应多个路径

实现的方法

  1. init(ServletConfig servletConfig)：初始化方法，在Servlet被创建时执行，只会执行一次
  2. getServletConfig()：获取ServletConfig对象
  3. service(ServletRequest servletRequest, ServletResponse servletResponse)：提供服务方法，每一次Servlet被访问，就会执行一次
  4. getServletInfo()：获取Servlet的一些信息，如版本、作者等等
  5. destroy()：销毁方法，在服务器正常关闭时，执行一次

细节讨论

  1. init方法只执行一次，说明Servlet在内存中只存在一个对象，Servlet是单例的，为了避免线程安全问题，应该尽量不要在Servlet中定义成员变量，即使定义了成员变量，也只用，而不要修改

  2. 默认情况下，第一次被访问时，Servlet被创建，可以配置web.xml改变Servlet的创建时机

  1. 只有服务器正常关闭时，destroy方法才会执行
  2. destroy方法是在Servlet被销毁之前执行的

GenericServlet和HttpServlet

实现Servlet接口就必须实现5个抽象方法，但实际上除了service方法，其他基本是用不到的，所以可以我们通常不直接实现Servlet接口，而是继承已经实现Servlet接口的子类

GenericServlet类是抽象类，它实现了Servlet接口，并对除service方法的其他4个方法进行了空实现，service方法保持抽象方法，这样一来，如果继承GenericServlet类，就不需要实现service之外的方法了

HttpServlet是GenericServlet的子类，我们每次实现service方法都要区分收到的是post请求还是get请求，根据不同的请求编写不同的代码，这样的重复工作在HttpServlet类中帮我们实现了，在HttpServlet类中，实现了service方法，并判断请求方式，如果是post请求就调用doPost方法，如果是get请求就调用doGet方法，实现请求的分发。这样一来，如果继承HttpServlet类，只需根据需要重写doPost方法和doGet方法就可以了
