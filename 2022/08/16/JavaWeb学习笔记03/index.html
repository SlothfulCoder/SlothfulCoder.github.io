JavaWeb学习笔记03

发表于 2022-08-16 更新于 2021-09-14

关键字：HTTP、请求消息、响应消息、Request、请求转发、Response、中文乱码问题、JavaBean、BeanUtils、重定向、相对路径、绝对路径、验证码图片生成、ServletContext、MIME、域对象、文件真实路径

1.Http

HTTP：Hyper Text Transfer Protocol 超文本传输协议，定义了客户端和服务器端通信时，发送数据的格式

特点

  1. 基于TCP/IP的高级协议
  2. 默认端口号:80
  3. 基于请求/响应模型的:一次请求对应一次响应
  4. 无状态的：每次请求之间相互独立，不能交互数据

历史版本

  • 1.0：每一次请求响应都会建立新的连接
  • 1.1：复用连接

请求消息

请求行

第一行是请求行

HTTP协议有7中请求方式，常用的有2种

  1. GET：
    • 请求参数在请求行中，在url后。
    • 请求的url长度有限制的
    • 不太安全
  2. POST：
    • 请求参数在请求体中
    • 请求的url长度没有限制的
    • 相对安全

请求头

第二行开始到空行之前是请求头，包含客户端浏览器给告诉服务器的一些信息

User-Agent：可以在服务器端获取该头的信息，解决浏览器的兼容性问题

Referer：可以用于防盗链和统计工作

请求空行

空行，就是用于分割POST请求的请求头，和请求体的

请求体

封装POST请求消息的请求参数的，只有POST请求方式，才有请求体

响应消息

服务器发送给客户端的数据

响应行

第一行是响应行

相应状态码：服务器告诉客户端浏览器本次请求和相应的一个状态，状态码都是3位数字组成

  1. 1xx：服务器接收客户端消息，但没有接收完成，等待一段时间后，发送1xx状态码
  2. 2xx：成功，代表：200
  3. 3xx：重定向，代表：302(重定向)，304(访问缓存)
  4. 4xx：客户端错误
    • 404：请求路径没有对应的资源
    • 405：请求方式没有对应的doXxx方法
  5. 5xx：服务器错误。代表：500(服务器内部出现异常)

响应头

  1. Content-Type：服务器告诉客户端本次响应体数据格式以及编码格式，常用text/html;charset=UTF-8
  2. Content-disposition：服务器告诉客户端以什么格式打开响应体数据
    • in-line：默认值，在当前页面内打开
    • attachment;filename=xxx：以附件形式打开响应体，文件下载

相应空行

空行，就是用于分割响应头和响应体的

响应体

传输给客户端的数据

2.Request

ServletRequest和HttpServletRequest都是接口，实际运行类型都是org.apache.catalina.connector.RequestFacade

获取请求消息数据

获取请求行数据

  1. String getMethod() ：获取请求方式——GET、POST
  2. String getContextPath()：获取虚拟目录——/servlet
  3. String getServletPath()：获取Servlet（资源）路径——/requestMethod
  4. String getQueryString()：获取get方式请求参数——name=tom&age=18
  5. String getRequestURI()：获取URI（统一资源标识符）——/servlet/requestMethod
  6. StringBuffer getRequestURL()：获取URL（统一资源定位符）——http://localhost:8080/servlet/requestMethod
  7. String getProtocol()：获取协议及版本——HTTP/1.1）
  8. String getRemoteAddr()：获取客户机IP地址（ipv6）——0:0:0:0:0:0:0:1

获取请求头数据

  1. String getHeader(String name)：通过请求头的名称获取请求头的值，如果没有获取到为null
  1. Enumeration<String> getHeaderNames()：获取所有的请求头名称，Enumeration的用法类似迭代器

获取请求体数据

  1. BufferedReader getReader()：获取字符输入流，只能操作字符数据
  1. ServletInputStream getInputStream()：获取字节输入流，可以操作所有类型数据

获取参数

上面虽然已经有getQueryString方法和getReader方法，但他们分别是用于获取get请求和post请求的参数，其实获取参数有get和post通用的方法

  1. String getParameter(String name)：根据参数名称获取参数值
  2. String[] getParameterValues(String name)：根据参数名称获取参数值的数组
  3. Enumeration<String> getParameterNames()：获取所有请求的参数名称
  4. Map<String,String[]> getParameterMap()：获取所有参数的map集合

使用getParameter后，可以用同一种方式获取get和post请求的参数，因此可以在doGet中调用doPost方法，然后只在doPost方法中实现逻辑

中文乱码问题

  1. get方式：tomcat 8 已经将get方式乱码问题解决了
  2. post方式：会乱码，在获取参数前，设置request的编码request.setCharacterEncoding(“utf-8”);

请求转发

有时一个Servlet不能完成我们的业务逻辑，我们需要另一个Servlet帮助，就需要把request和response转发到另一个Servlet

特点

  1. 浏览器地址栏路径不发生变化
  2. 只能转发到当前服务器内部资源中
  3. 转发是一次请求
  4. 请求转发调用的是doPost方法

请求转发步骤

  1. RequestDispatcher getRequestDispatcher(String path)：通过request对象获取请求转发器对象
  2. forward(ServletRequest request, ServletResponse response)：使用RequestDispatcher对象来进行转发

请求转发相当于调用了另一个Servlet的service方法，这个service方法执行结束后会回到原来的service方法

共享数据

request域：代表一次请求的范围，一般用于请求转发的多个资源中共享数据

请求转发有时不止需要把request和response传过去，还需要传递过去一些数据

  1. void setAttribute(String name,Object obj)：把一个对象存入request
  2. Object getAttitude(String name)：通过键获取值
  3. void removeAttribute(String name)：通过键移除键值对

3.JavaBean

标准的Java类，一般用于对应数据库中的一行数据

要求

  1. 类必须被public修饰
  2. 必须提供空参的构造器
  3. 成员变量必须使用private修饰
  4. 提供公共setter和getter方法

4.BeanUtils工具类

导入jar包：commons-beanutils-1.8.0.jar 和 commons-logging-1.2.jar

我们从request中用getParameter方法一条一条的取出数据，再封装到一个JavaBean中太过麻烦，使用BeanUtils工具类可以帮助我们完成这些操作

  1. populate(Object obj , Map map)：将map集合的键值对信息，封装到对应的JavaBean对象中，map集合一般来自request的getParameterMap方法
  1. setProperty()：给JavaBean的属性赋值
  1. getProperty()：获取JavaBean的属性值

5.Response

ServletResponse和HttpServletResponse都是接口，实际运行类型都是org.apache.catalina.connector.ResponseFacade

设置响应消息

  1. setStatus(int sc) ：设置状态码
  2. 设置响应头
    • setHeader(String name, String value) ：设置任意响应头
    • setContentType(String var1)：设置要发送的文件类型，取值可以查询tomcat的主web.xml或用ServletContext的getMimeType(String fileName) 方法获取
      • 取值如果是text/html，防止中文乱码可以加上编码方式：text/html;charset=utf-8
  3. 设置响应体
    1. 获取输出流
      • PrintWriter getWriter()：字符输出流
      • ServletOutputStream getOutputStream()：字节输出流
    2. 使用输出流，将数据输出到客户端浏览器

输出字符数据到浏览器

  1. 获取字符输出流
  2. 使用字符输出流，将数据输出到客户端浏览器

注意：PrintWriter从是response获取的，一次响应结束之后，response会自动被销毁，这个PrintWriter流也会被关闭掉，所以不需要刷新就能写出到缓冲区

中文乱码问题

浏览器会跟随操作系统的字符集解码，一般是gbk，但是从response获取到的PrintWriter流的默认编码是ISO-8859-1，发送的字符编码和解码方式不一致，就会导致乱码问题

解决方法：在获取PrintWriter之前，设置response的编码方式，并告诉浏览器希望的解码方式（设置响应头content-type）

输出字节数据到浏览器

一般是发送文件

  1. 获取字节输出流
  2. 输出数据

中文文件名问题

响应体为一个文件时，需要设置响应头content-disposition为attachment;filename=xxx，其中xxx是文件名，但是不同浏览器处理文件名的方式不同，为了保证不同浏览器解析到的文件名相同，应该判断当前请求来自什么浏览器，对文件名做出相应的处理

重定向

告诉客户端，让客户端重新请求另一个资源

重定向代码

重定向方法：sendRedirect(String var1)

重定向的特点

注意和请求转发的区别

  1. 地址栏发生变化
  2. 重定向可以访问其他站点(服务器)的资源
  3. 重定向是两次请求。不能使用request对象来共享数据

6.相对路径和绝对路径

  1. 相对路径：根据当前资源和目标资源之间的相对位置关系，找到目标资源
  2. 绝对路径：http://localhost/day15/responseDemo2可以省略ip地址(域名)，从虚拟目录开始写：/day15/responseDemo2
    • 请求从浏览器发出，需要加虚拟目录；请求从服务器发出（请求转发），不需要加虚拟目录

8.ServletContext

ServletContext代表整个web应用，可以和程序的容器(服务器)来通信，一个项目只有一个ServletContext对象

获取ServletContext

每次获取到的都是同一个ServletContext对象

  1. 通过request对象获取
  1. 通过HttpServlet获取

获取MIME类型

MIME类型：在互联网通信过程中定义的一种文件数据类型

MIME类型格式：大类型/小类型，如text/html，image/jpeg

获取方法MIME类型方法

String getMimeType(String fileName)：传入文件名，返回这类型文件的MIME类型，原理是截取文件名的后缀名，根据后缀名到tomcat的主配置文件/conf/web.xml中匹配

域对象

域对象就是把数据存入ServletContext对象，由于Servlet整个服务器只有一个ServletContext对象，且每个Servlet都能获取到它，所以存到ServletContext对象的数据是整个服务器共享的。

这些数据在内存中存留时间长，直到服务器关闭这些数据才会被销毁，因此要谨慎使用

相关方法

  1. setAttribute(String name,Object value)：放入数据
  2. getAttribute(String name)：获取数据
  3. removeAttribute(String name)：删除数据

获取文件的真实（服务器）路径

IDEA会为每一个tomcat部署的项目单独建立一份配置文件，配置文件中写的路径就是服务器的真实(服务器)路径，SerletContext提供了getRealPath方法，用于获取文件的真实路径

String getRealPath(String path)：该方法的效果是把服务器真实路径和参数拼接在一起，形成新的字符串返回
