获得Restful风格的参数

Restful是一种软件架构风格、设计风格，而不是标准，只是提供了一组设计原则和约束条件。主要用于客户端和服务器交互类的软件，基于这个风格设计的软件可以更简洁，更有层次，更易于实现缓存机制等。

Restful风格的请求是使用“url+请求方式”表示一次请求目的的，HTTP 协议里面四个表示操作方式的动词如下：

  1. GET：用于获取资源，/user/1，得到id = 1 的 user
  2. POST：用于新建资源，/user/1，删除id = 1 的 user
  3. PUT：用于更新资源，/user/1，更新id = 1 的 user
  4. DELETE：用于删除资源，/user，新增user

url地址/save7/lily中的lily就是要获得的请求参数，在SpringMVC中可以使用占位符进行参数绑定。地址/save7/lily可以写成/save7/{name}，占位符{name}对应的就是lily的值。在业务方法中我们可以使用@PathVariable注解进行占位符的匹配获取工作。

请求参数和方法形参名不一致

当请求参数的名称与Controller的业务方法参数名称不一致时，就需要通过@RequestParam参数绑定注解显示的绑定。

@RequestParam常用属性

  • value：与请求参数名称匹配
  • required：此在指定的请求参数是否必须存在，默认值时true，代表提交时如果没有此参数就报错
  • defaultValue：当没有指定参数时，则使用指定的默认值赋值

自定义类型转换器
